Quantum algorithms

Historical note

The discovery of efficient quantum algorithms marks one of the great intellectual achievements of the late twentieth century. In 1994, Peter Shor  showed that a quantum computer could factor integers in polynomial time, threatening the security of RSA encryption. Two years later, Lov Grover  demonstrated a quadratic speedup for unstructured search. In 2009, Harrow, Hassidim, and Lloyd  showed that quantum computers could solve certain linear systems exponentially faster than any known classical algorithm. These three results continue to form the conceptual foundation for most quantum algorithms discovered since.

Overview

The two most celebrated quantum algorithms are Shor’s factoring algorithm (1994) and Grover’s search algorithm (1996). A third important example, possibly more relevant for near-term applications, is the Harrow-Hassidim-Lloyd (HHL) linear-systems algorithm (2008). For a comprehensive catalogue of quantum algorithms, see the Quantum Algorithm Zoo1.

Although there are now very many quantum algorithms, they can often be traced back fairly directly to one of these three. It is fair to say that our understanding of the discovery and design of quantum algorithms is still in its infancy. The field of quantum software engineering, which would presumably supply robust design practices and patterns, has not yet been fully developed.

Physical intuition

A quantum algorithm provides a quantum advantage (or quantum speedup) when it solves a computational problem using asymptotically fewer resources (time, queries, or gates) than any known classical algorithm for the same problem. Advantages can be:

  • Exponential: $O\!\left(\poly{n}\right)$ vs. 2Ω(n) (e.g., Shor’s algorithm for factoring).

  • Polynomial: $O\!\left(\sqrt{N}\right)$ vs. O (N) (e.g., Grover’s search).

  • Provable: demonstrated via lower bounds in a query or communication model.

  • Heuristic: observed empirically but without rigorous worst-case guarantees (e.g., variational quantum algorithms).

Understanding which problems admit quantum speedups, and why, remains one of the central open questions in quantum computing.

The query model

To explain the query setting, consider an N-bit input x = (x0,…,xN − 1) ∈ {0, 1}×N. Usually we take N = 2n, so that bit xj can be addressed using an n-bit index j. One can think of the input as an N-bit memory which we can access at any point of our choice (a “Random Access Memory”, or RAM).

Memory accesses are modelled via “black-box” unitary operations:

Definition.

The standard query oracle for an N-bit string x ∈ {0, 1}×N is the unitary on n + 1 qubits defined by $$\boxed{O_x:|j,b\rangle \mapsto |j,b\oplus x_j\rangle,}$$ where j ∈ {0, 1}×n, b ∈ {0, 1}, and ⊕ denotes addition modulo 2. The first n qubits are the address register and the (n+1)st qubit is the target bit.

As a matrix, Ox is a permutation matrix and hence unitary. Because Ox is a quantum operation, a quantum computer can apply it on a superposition of addresses j—something a classical computer cannot do.

Definition.

Given the ability to make a standard query, we can also make a phase query by setting the target bit to $|-angle = \frac{1}{\sqrt{2}}(|0angle - |1angle) = H|1angle$: Ox|j⟩| − angle = (−1)xj|j⟩| − angle. The corresponding n-qubit unitary (ignoring the ancilla in the fixed state | − angle) is $$\boxed{O_{x,\pm}|j\rangle = (-1)^{x_j}|j\rangle.}$$

Physical intuition

The technique of placing the answer in the phase rather than in a register is called phase kickback. It is a fundamental trick in quantum computing: the ancilla qubit | − angle is an eigenstate of X with eigenvalue  − 1, so the oracle’s action “kicks back” the function value xj as a phase (−1)xj onto the address register. This trick underlies the Deutsch-Jozsa algorithm, Grover’s algorithm, and quantum phase estimation.

The Deutsch-Jozsa algorithm

Key result

For N = 2n, we are given x ∈ {0, 1}×N such that either:

  1. all xj have the same value (“constant”); or

  2. half (N/2) of the xj are 0 and the other half are 1 (“balanced”).

The objective is to determine which situation holds.

Algorithm

  1. Initialise an n-qubit state in |0angle⊗n.

  2. Apply a Hadamard gate to each qubit: H⊗n.

  3. Apply Ox, ± to the qubits.

  4. Apply H⊗n again.

  5. Measure all qubits in the computational basis.

The complete quantum algorithm is the unitary $$\boxed{U_{\mathrm{DJ}} \equiv H^{\otimes n}O_{x,\pm} H^{\otimes n}.}$$

The circuit for the Deutsch-Jozsa algorithm is:

We follow the state of the quantum computer through each step. Initially the state is |0angle⊗n. After the first Hadamard transform: $$H^{\otimes n}|0angle^{\otimes n} = \frac{1}{\sqrt{2^n}}\sum_{j\in \{0,1\}^{\times n}} |j\rangle.$$ After the phase oracle: $$O_{x,\pm} H^{\otimes n}|0angle^{\otimes n} = \frac{1}{\sqrt{2^n}}\sum_{j\in \{0,1\}^{\times n}} (-1)^{x_j}|j\rangle.$$ After the second Hadamard: $$H^{\otimes n}O_{x,\pm} H^{\otimes n}|0angle^{\otimes n} = \frac{1}{2^n}\sum_{j\in \{0,1\}^{\times n}}(-1)^{x_j} \sum_{k\in \{0,1\}^{\times n}} (-1)^{j\cdot k}|k\rangle,$$ where $j\cdot k \equiv \sum_{l=1}^n j_l k_l$. Since j ⋅ 0 = 0 for all j, the amplitude of the |0angle⊗n state in the final superposition is $$\frac{1}{2^n}\sum_{j\in \{0,1\}^{\times n}}(-1)^{x_j} = \begin{cases} 1 & \text{if $x_j=0$ for all $j$ (constant $0$),} \\ -1 & \text{if $x_j=1$ for all $j$ (constant $1$),} \\ 0 & \text{if $x$ is balanced.} \end{cases}$$ Hence the final measurement yields |0angle⊗n with certainty if x is constant, and yields some other state if x is balanced.

Key result

The Deutsch-Jozsa problem can be solved with certainty using only 1 quantum query and O (n) other operations. Any classical deterministic algorithm needs at least N/2 + 1 queries.

Remark.

The quantum-classical separation for Deutsch-Jozsa only holds for deterministic algorithms. A classical randomised algorithm can solve this problem efficiently: query x at two random positions; output “constant” if those bits are the same and “balanced” if they differ. This succeeds with probability at least 1/2 when x is balanced.

Grover’s search algorithm

The second most important quantum algorithm after Shor’s is Grover’s search algorithm . It provides a quadratic speedup—not exponential—but is far more widely applicable.

The search problem

Key result

For N = 2n, given x ∈ {0, 1}×N, find a j such that xj = 1 (or output “no solutions” if no such j exists). We denote the number of solutions by t (the Hamming weight of x).

This problem models searching an N-slot unordered database. Classically, a randomised algorithm needs Θ(N) queries. Grover’s algorithm solves it in $O\!\left(\sqrt{N}\right)$ queries and $O\!\left(\sqrt{N}\log N\right)$ other gates.

The Grover iterate

We again exploit the phase oracle Ox, ±|j⟩=(−1)xj|j⟩. Define the zero-reflection: $$R_0|j\rangle \equiv \begin{cases} |\mathbf{0}\rangle & \text{if } j= \mathbf{0}, \\ -|j\rangle & \text{if } j\not= \mathbf{0}. \end{cases}$$ That is, R0 = 2|0⟩⟨0| − 𝕀.

Definition.

The Grover iterate (or Grover diffusion operator) is $$\boxed{\mathcal{G} \equiv H^{\otimes n}R_0 H^{\otimes n}O_{x,\pm} = \left(2|U\rangle\langle U| - \mathbb{I}\right)O_{x,\pm},}$$ where $|U\rangle = H^{\otimes n}|0angle^{\otimes n} = \frac{1}{\sqrt{N}}\sum_j|j\rangle$ is the uniform superposition. A single Grover iterate uses one query and O (logN) other gates.

Algorithm

  1. Prepare the uniform superposition |U⟩=H⊗n|0angle⊗n.

  2. Apply the Grover iterate 𝒢 a total of k times (for some k to be determined).

  3. Measure the final state in the computational basis.

  4. Verify whether the result j satisfies xj = 1.

Geometric analysis

There is an elegant geometric argument for the correctness of Grover’s algorithm. Define the “good” and “bad” states: $$|G\rangle \equiv \frac{1}{\sqrt{t}}\sum_{\substack{j:\\x_j=1}}|j\rangle, \qquad |B\rangle \equiv \frac{1}{\sqrt{N-t}}\sum_{\substack{j:\\x_j=0}}|j\rangle.$$ Then the uniform state can be written as $$|U\rangle = \sin(\theta)|G\rangle + \cos(\theta)|B\rangle, \qquad \text{where } \theta = \sin^{-1}\!\left(\sqrt{t/N}\right).$$

Physical intuition

The Grover iterate 𝒢 is the product of two reflections in the 2-dimensional real plane spanned by |G⟩ and |B⟩:

  1. Ox, ± reflects through |B⟩ (i.e., Ox, ± = 𝕀 − 2|G⟩⟨G| restricted to this plane).

  2. 2|U⟩⟨U| − 𝕀 reflects through |U⟩.

Each application of 𝒢 rotates the state by an angle 2θ towards |G⟩.

After k applications of 𝒢, the state becomes 𝒢k|U⟩=sin((2k+1)θ)|G⟩ + cos ((2k+1)θ)|B⟩. The probability of observing a solution upon measurement is Pk = sin2((2k+1)θ).

We want Pk close to 1. Setting $\widetilde{k} = \frac{\pi}{4\theta} - \frac{1}{2}$ gives (2k̃+1)θ = π/2 and hence Pk̃ = 1.

Example.

If t = N/4 then θ = π/6 and k̃ = 1. Thus a single Grover iteration suffices to find a solution with certainty.

In general k̃ is not an integer. Choosing k to be the nearest integer, we bound the failure probability: $$\begin{split} 1-P_k &= \cos^2\bigl((2k+1)\theta\bigr) = \sin^2\bigl(2(k-\widetilde{k})\theta\bigr) \\ &\le \sin^2(\theta) = \frac{t}{N}, \end{split}$$ where we used $|k-\widetilde{k}|\le \frac{1}{2}$. Since $\sin^{-1}(\sqrt{t/N})\ge \sqrt{t/N}$, the number of queries is $$\boxed{k\le \frac{\pi}{4}\sqrt{\frac{N}{t}}.}$$

Variants and extensions of Grover’s algorithm

  • Exact search: If t is known exactly, the algorithm can be tweaked to end up in exactly |G⟩ by adjusting θ.

  • Unknown t: If we do not know t, running the algorithm with exponentially increasing guesses for k still gives an expected query count of $O\!\left(\sqrt{N/t}\right)$. If t = 0, this is detected by verifying xj for the output j.

  • Known lower bound: If τ ≤ t is a known lower bound, the algorithm uses an expected $O\!\left(\sqrt{N/\tau}\right)$ queries.

  • Boosted error reduction: To reduce the failure probability to ϵ, one can use $O\!\left(\sqrt{N \log(1/\epsilon)}\right)$ queries—note the log (1/ϵ) is inside the square root, which is better than naive repetition giving $O\!\left(\sqrt{N}\log(1/\epsilon)\right)$.

Remark.

Grover’s $O\!\left(\sqrt{N}\right)$ query complexity is optimal: Bennett, Bernstein, Brassard, and Vazirani (1997) proved that any quantum algorithm for unstructured search requires $\Omega(\sqrt{N})$ queries. This is one of the few known tight quantum lower bounds.
